# Task Plan: Premium vendor workspace

> Source spec: `docs/flows/vendor-dashboard/spec.md`

## Tasks

1. **Compact shell and overview** — remove duplicate demo/marketing treatment, tighten the sidebar, and replace stat cards with a useful filter summary. Done when jobs begin above the fold.
2. **Premium queue** — add consistent status/date metadata, a declined filter, flatter rows, and desktop-only internal scrolling. Done when work can be scanned without opening every record.
3. **Premium record** — flatten nested cards, put actions in the record header, and move the horizontal Amanah trail directly below it. Done when the next action and lifecycle are immediately visible.
4. **Responsive master/detail** — use the existing client state to switch between list and detail below `xl`. Done when selection and return require one action each.
5. **Reports and states** — align reports, empty, declined, proof, and success states with the operational system. Done when every existing branch still works.
6. **QA and release** — verify interactions, keyboard focus, responsive layouts, public/admin regressions, lint, tests, build, GitHub, and Vercel. Done when every acceptance criterion passes live.

## Checkpoint

The user explicitly requested a Mobbin-led premium redesign and authorised design-system changes. That is treated as sign-off for this scoped visual/UX pass; authentication and persistent backend work remain separate phases.
