# Task Plan: Public site v1

> Source spec: `docs/flows/public-site-v1/spec.md`

## Tasks

1. **Brand foundation** — add the supplied seal as a public asset; replace CSS tokens with the approved palette; retain the existing installed typography. Done when the shell renders the official identity with accessible contrast.
2. **Navigation and footer** — rebuild the utility bar, desktop navigation, native mobile menu, service links, telephone link and professional social placeholders. Done when every public route is reachable on desktop and mobile.
3. **Homepage** — adapt the TravelConnect/Global Ehsan section rhythm into the As-Sābiqūn hero, four service cards, three-step Amanah trail, accountability preview and closing CTA. Done when the page communicates the complete public journey without invented proof.
4. **Services index** — add `/services` with Korban, Wakaf Water Pump, Wakaf Quran and Food for Orphans. Done when each card leads to the appropriate existing service route.
5. **Supporting pages** — bring About, Korban, Wakaf and Contact into the same hierarchy; add mission/vision/team-focus content and “what you receive” blocks where relevant. Done when the pages feel like one product rather than separate templates.
6. **Responsive and accessibility QA** — test desktop and mobile navigation, keyboard focus, contrast, overflow, headings and form labels. Done when public pages work at 390px and desktop widths with no console errors.
7. **Verification and release** — run lint, tests and production build; inspect the final diff; commit only Part 1 files; push and verify the Vercel deployment. Done when the live public routes show the approved design and `/designs` remains intact.

## Checkpoint

Approved and implemented. The public routes, preserved dashboard, and `/designs` were visually checked at desktop and mobile widths before release.
